# Zotero Linker 1.0.1

## 中文

本次发布修复了一个会影响 Word 文档状态的行为。

### 修复原因

旧版本会在 Word 光标或选择区域变化时自动刷新超链接样式。这个自动刷新可能在用户没有点击插件功能、只是打开或浏览文档时修改文档内部样式，导致 Word 关闭文档时提示“是否保存修改”。同时，这些自动样式写入也可能进入 Word 的撤销栈，使用户粘贴或编辑文字后需要多次按 `Ctrl + Z` 才能撤销到预期状态。

### 本次改动

- 移除了插件启动后的 `WindowSelectionChange` 自动监听写入逻辑。
- 插件不再因为打开文档、移动光标或改变选择区域而主动修改文档。
- 文档内容和样式只会在用户点击 `Link Citations`、`Remove Links`、`Repair Formatting` 或保存选项后按预期更新。

### 适用用户

建议所有已经安装 `1.0.0` 的用户升级，尤其是遇到“未操作也提示保存”或“撤销次数异常变多”的用户。

## English

This release fixes behavior that could unexpectedly mark Word documents as modified.

### Why this was fixed

The previous version refreshed hyperlink styles automatically when the Word cursor or selection changed. That automatic refresh could modify document styles even when the user had not clicked any Zotero Linker command, causing Word to ask whether to save changes when closing an otherwise untouched document. These automatic style writes could also enter Word's undo stack, so users might need to press `Ctrl + Z` multiple times after pasting or editing text.

### What changed

- Removed the automatic `WindowSelectionChange` document-writing handler.
- The add-in no longer changes the document simply because a document is opened, the cursor moves, or the selection changes.
- Document content and formatting are now changed only after explicit user actions such as `Link Citations`, `Remove Links`, `Repair Formatting`, or saving options.

### Who should update

All users on `1.0.0` should update, especially anyone seeing unexpected save prompts or unusually long undo chains.
